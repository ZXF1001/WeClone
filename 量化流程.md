1. 执行以下命令合并模型
```sh
llamafactory-cli export \
  --model_name_or_path /autodl-fs/data/models/Qwen3-32B \
  --adapter_name_or_path /autodl-fs/data/output/model_output-zxf-260220-Qwen3-32B \
  --template qwen3 \
  --finetuning_type lora \
  --export_dir ./zxf_qwen3 \
  --export_size 5 \
  --export_device auto \
  --export_legacy_format false
```

2. 下载llama.cpp
```
git clone
```

3. 安装依赖
```
pip install -r requirements/requirements-convert_hf_to_gguf.txt
```

4. 合并为fp16精度的gguf
```
python llama.cpp/convert_hf_to_gguf.py ~/autodl-tmp/WeClone/zxf_qwen3 \
      --outfile ~/autodl-tmp/WeClone/zxf.gguf \
      --outtype f16
```

5. 编译llama-quantize工具

6. 量化为q4_k_m精度的gguf
```
./build/bin/llama-quantize ~/autodl-tmp/WeClone/zxf.gguf ~/autodl-tmp/WeClone/zxf_q4_k_m.gguf q4_k_m
```

7. 下载下来或者上传到魔塔保存，以后用于ollama部署

8. ollama部署流程：先找一个Modelfile（从合并后的safetensors文件夹里面找一个base，然后加入
```
FROM ./mxx_q5_k_m.gguf
...

# 温度和发散度
PARAMETER temperature 0.7
PARAMETER top_p 0.8

SYSTEM """请你扮演一名人类，不要说自己是人工智能"""
```

9. 然后`ollama create mxx_q5_k_m -f .\Modelfile`

10. ollama run mxx_q5_k_m --think=false
